2. Git 개요

   - 버전 관리 시스템임
   - 리누스 토르발스가 만듬 ( 리눅스 창시자 )

   - 특징:
      로컬 저장소 및 원격 저장소 생성: git init 명령어
      로컬 저장소에 버전관리할 파일 생성 및 추가 : git add 명령어 ( 아직까지는 버전 생성 안됨 )
      버전관리할 파일의 버전 생성 :  git commit 명령어 + 커밋 메시지 반드시 설정( ex. "update 코드 추가" )
      버전 히스토리 확인 : git log 명령어
      버전간에 비교 : git diff 명령어
      특정버전으로 되돌아기 : git reset 명령어 , git revert 명령어
      새로운 브랜치 생성 ( 이전 브랜치에 영향을 미치지 않음 ) : git branch 브랜치명
      브랜치 병합: git merge 명령어
      로컬저장소에 브랜치를 원격 저장소에 보내기 가능: git push 명령어

3. Git이 사용하는 기본 텍스트 편집기 설정

  https://git-scm.com/book/en/v2/Appendix-C%3A-Git-Commands-Setup-and-Config 가이드

  git config --global core.editor "code --wait" <== git commit 명령어만 사용하면 자동으로 VSC 오픈됨


4. 전역 사용자 등록

   -git bash 에서 작업
   -------------------------
    ls      # 디렉터리 목록보기
    ls -al  # 자세하게 디렉터리 목록보기

    clear   # 화면 지우기

    pwd     # 현재 위치
    
    mkdir 디렉터리명  #디렉터리 생성

    cd  디렉터리명   #디렉터리 변경 , 몇글자만 입력하고 tab 활용
    cd  ..        #상위 디렉터리로 변경

    rm -rf 디렉터리명  # 디렉터리 강제 삭제
    rm 파일명 파일명2 ..        # 파일 삭제

    touch 파일명     # 파일생성

    echo 문자열      # 문자열이 콘솔에 출력
    
    echo 문자열 >> 파일명     # 콘솔에 출력되는 문자열을 파일에 추가(append, 새로운 줄에 추가됨), redirection 기능
    echo -n 문자열 >> 파일명  # 콘솔에 출력되는 문자열을 파일에 추가(append, 같은 줄에 추가됨),
    echo 문자열 >  파일명  # 콘솔에 출력되는 문자열을 파일에 덮어쓰기

    cat 파일명   # 파일내용 보기

##############
 로컬 저장소 생성 및 관리
#############

* git 관련 상태 3가지

            git add test.txt                     git commit
  작업디렉터리   --------------> 스테이지(staging area) -----------------> 로컬 저장소
 ex.git_tutorial                 test.txt                            test.txt 버전으로 관리됨.
                                                                     (해시값관리: 40글자 )
   test.txt(관리대상)
   a.txt
   b.txt


1. 저장소 생성
 
  명령어: git init
  
  실습순서:
    
    $ mkdir git_tutorial
    $ cd git_tutorial
    $ git init   # 저장소 생성, .git 폴더가 생성됨

2. 저장소 상태확인
    $ git status # 3가지 상태 확인
 
        On branch main   # 현재 main 브랜치에 있음
        No commits yet   # 아직 커밋한 파일이 없다.
        nothing to commit (create/copy files and use "git add" to track) # 커밋할 파일도 없다.

  
   # 파일 생성
    touch hello.txt

   $ git status
        On branch main # 현재 main 브랜치에 있음
	No commits yet # 아직 커밋한 파일이 없다.
	Untracked files: # git이 한번도 버전관리 되지 않은 파일의미
	  (use "git add <file>..." to include in what will be committed)
		hello.txt
	nothing added to commit but untracked files present (use "git add" to track)

3. 추적시킬 파일 등록 ( 스테이징, 스테이지에 올린다. index 등록한다 라고 함 )

  문법: 
      git add 파일명, 파일명,... # 지정된 파일명 스테이징으로 
      git add  .  # 현재 디렉터리 모든 파일을 스테이징으로
            
	      git add hello.txt
  작업디렉터리   --------------> 스테이지(staging area) -----------------> 로컬 저장소
 ex.git_tutorial                 hello.txt                        
   hello.txt 
  (untracked file)

   실습:
     $ git add hello.txt
     $ git status
      
        On branch main # 현재 main 브랜치에 있음
	No commits yet # 아직 커밋한 파일이 없다.
	Changes to be committed: # hello.txt 앞으로 버전관리할 것임.
	  (use "git rm --cached <file>..." to unstage)
		new file:   hello.txt

     #unstage
       $ git rm --cached hello.txt
       $ git status

4. 버전 생성

   문법: 
       git commit + 커밋 메시지 지정

       git commit -m 커밋메시지   <== 전역사용자 정보가 같이 추가됨 


   	      git add hello.txt
  작업디렉터리   --------------> 스테이지(staging area) -----------------> 로컬 저장소
 ex.git_tutorial                 hello.txt                            hello.txt 버전 생성
   hello.txt 
  (untracked file)

   실습:
    
     $ git commit -m "first commit"
     $ git status
     $ git log
